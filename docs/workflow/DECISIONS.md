# Phase 1 의사결정 기록 (확정)

> 업데이트: 2026-02-27
> 목적: PoC 부트스트랩에서 즉시 구현 가능한 기준선을 고정하고, 애매한 항목은 실무 기본값으로 빠르게 결정한다.

## 1) 확정 결정 목록 (Q1~Q10)

| 질문 | 확정안 | 간단 근거 |
|---|---|---|
| Q1. MVP 보안 범위 | **C: SAST + SCA + Secret 탐지** | 실제 개발팀에서 가장 먼저 체감하는 리스크(코드 취약점/의존성/키 유출)를 동시에 커버 |
| Q2. 우선 가치 | **A + B: 가격 + 간편함** | 스타트업 도입 장벽의 핵심은 비용과 운영 복잡도이므로 초기 제품 메시지와 UX를 여기에 집중 |
| Q3. MVP 핵심 기능 | **B: 스캔 + 대시보드 + CI 연동** | “실행→가시화→개발 플로우 연결”의 최소 가치 사슬을 한 번에 검증 가능 |
| Q4. 백엔드 언어/프레임워크 | **TypeScript + Fastify 고정** | 이번 스프린트 목표가 PoC 속도/생산성이므로 구현 및 인력 수급이 유리한 조합 선택 |
| Q5. 프론트엔드 | **A: Next.js** | App Router 기반으로 화면/라우팅 부트스트랩이 빠르고, 향후 배포/확장도 용이 |
| Q6. DB | **PostgreSQL** | 운영 안정성, 생태계 성숙도, 분석/리포팅 확장성 측면에서 안전한 기본값 |
| Q7. CI 연동 방식 | **A: GitHub App** | GitHub 중심 팀에서 권한/설치/자동화 UX가 가장 일관적 |
| Q8. 언어 지원 우선순위 | **C: JS/TS + Python 우선 지원** | 스타트업/초기 SaaS에서 점유율이 높아 초기 고객 적합성 확보에 유리 |
| Q9. 타겟 팀 크기 | **A: 초기 1~5명 팀** | 온보딩/운영 단순화 요구가 가장 강하고, PoC 피드백 루프가 빠름 |
| Q10. 인증 | **C: Google SSO** | 스타트업의 조직 계정 환경과 친화적이며 사용자 온보딩 마찰이 낮음 |

### Q4 보완 메모 (성능 리스크 대응)

- 현재 스프린트는 **TypeScript + Fastify**로 고정한다.
- 이후 스캔 처리량 증가/지연 문제가 발생하면, **Go 기반 스캔 워커 분리**(비동기 잡 큐) 옵션을 Phase 2~3 기술 검토 항목으로 유지한다.

## 2) 미결정 항목

아래 항목은 이번 Phase 1 범위 밖이며, 구현 직전 단계에서 확정한다.

1. SAST/SCA/Secret 엔진 구체 제품 선정 (예: Semgrep, Trivy, Gitleaks 조합)
2. 취약점 심각도 정책(차단 기준, False Positive 처리 규칙)
3. GitHub App 권한 스코프 최소화 설계
4. Google SSO 세션 전략(세션 저장소, 만료/재인증 정책)
5. 대시보드 지표 정의(MTTR, 취약점 추세, 저장 기간)

## 3) 리스크 및 완화 전략

| 리스크 | 영향 | 완화 전략 |
|---|---|---|
| 스캔 범위 확대로 처리 시간 증가 | CI 파이프라인 지연, 사용자 이탈 | 비동기 실행 + 큐잉 + 캐시, 필요 시 Go 워커 분리 |
| 초기 도구 선택 미스매치 | 정확도/속도 저하, 운영 피로 | 엔진 추상화 레이어로 교체 비용 최소화, 2주 단위 벤치마크 |
| Google SSO만으로는 일부 고객 요구 미충족 | 엔터프라이즈 전환 시 제약 | OAuth Provider 확장 가능한 인터페이스로 설계 |
| GitHub 의존도 집중 | GitLab 사용자 확장 지연 | CI 어댑터 인터페이스 선설계 후 GitLab 커넥터를 후속 추가 |
| 소규모 팀 리소스 한계 | 문서/테스트/기능 품질 불균형 | 범위 고정(Phase 목표), 자동화 테스트 우선, TODO 명시 운영 |

## 4) 이번 스프린트 구현 범위 요약

- API: `/health`, `/api/v1/scans(501 stub)`
- Web: Next.js App Router 최소 화면 + 확정 의사결정 표시
- Workspace: pnpm 모노레포, 공통 TS 설정, 로컬 PostgreSQL(docker-compose)
- 검증: API 테스트/타입체크를 pnpm 기반으로 실행
